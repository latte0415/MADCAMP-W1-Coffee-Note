# Coffee Note 디자인 가이드

## 목적
이 문서는 Coffee Note 애플리케이션의 디자인 시스템과 UI/UX 가이드라인을 정의합니다. 개발 및 디자인 작업 시 참고해야 할 기준을 제시합니다.

## 디자인 철학
Coffee Note는 사용자가 커피에 대한 감각적 경험을 체계적으로 기록하고 관리할 수 있도록 돕는 애플리케이션입니다. 디자인은 **명확성(Clarity)**, **일관성(Consistency)**, **사용성(Usability)**을 핵심 가치로 합니다.

---

## 디자인 시스템

### 색상 팔레트

#### 주요 색상
- **Primary Dark**: `#2B1E1A`
  - 사용: 주요 텍스트, 버튼 배경, 강조 요소
  - 예: 제목 텍스트, 버튼 배경, 해시태그 배경
  
- **Background**: `#FFFFFF`
  - 사용: 화면 배경, 카드 배경
  - 예: 전체 화면 배경, 입력 필드 배경

- **Border**: `rgba(90, 58, 46, 0.3)`
  - 사용: 입력 필드 테두리, 구분선
  - 예: 텍스트 입력창 테두리, 필터 영역 테두리

#### 텍스트 색상
- **Primary Text**: `#2B1E1A` (Primary Dark와 동일)
- **White Text**: `#FFFFFF` (다크 배경 위 텍스트)

### 타이포그래피

#### 폰트 패밀리
- **Font Family**: `NanumSquareOTF` (나눔스퀘어)
  - 모든 텍스트에 적용
  - 한글 가독성 최적화

#### 폰트 크기
- **Large Title**: 50px, Weight: 700 (Bold)
  - 사용: 화면 제목, 중요한 정보 표시
  - 예: 메뉴명, 카페명 (큰 제목)
  - Line Height: 57px

- **Body Text**: 30px, Weight: 400 (Regular)
  - 사용: 본문 텍스트, 일반적인 정보
  - 예: 설명 텍스트, 버튼 텍스트, 입력 필드 텍스트
  - Line Height: 34px
  - Letter Spacing: 0.1em

### 간격 및 레이아웃

#### Border Radius
- **Large Radius**: 20px
  - 사용: 버튼, 카드, 해시태그 배경
  - 예: "기록하기" 버튼, 해시태그 배경

- **Small Radius**: 10px
  - 사용: 입력 필드, 작은 컨테이너
  - 예: 텍스트 입력창, 필터 영역

#### 화면 크기 기준
- 디자인 기준: 1080px × 2400px (모바일 세로 방향)
- 좌우 여백: 49px (컨텐츠 시작 위치)
- 컴포넌트 너비: 972px (1080px - 49px × 2)

## 화면별 디자인 가이드

### 1. Creation (1-1-0): 생성 팝업 기본 화면

#### 구성 요소
- **뒤로가기 버튼**: 왼쪽 상단, 이전 화면으로 이동
- **이미지 추가하기**: 이미지 업로드 기능
- **텍스트 입력창**: 
  - 메뉴명, 카페명, 날짜, 한줄평
  - 배경: `#FFFFFF`
  - 테두리: `rgba(90, 58, 46, 0.3)`, 2px solid
  - Border Radius: 10px
- **슬라이더**: 
  - 산미, 바디, 쓴맛 (1~10 단위)
  - 각 특성에 대한 시각적 표현 필요
- **상세정보 추가하기 체크박스**: 
  - 체크 시 화면 확장 (scroll 가능)
  - 자세한 내용은 creation_spread 참고
- **기록하기 버튼**: 
  - 배경: `#2B1E1A`
  - 텍스트: `#FFFFFF`
  - Border Radius: 20px
  - 입력 정보를 바탕으로 createNote 실행

#### 디자인 주의사항
- 팝업 형태로 구현 (depth 증가 없음)
- 스크롤 가능한 영역 제공
- 입력 필드 간 적절한 간격 유지

### 2. Creation Spread (1-1-0): 상세 정보 추가 화면

#### 기본 구성
- Creation (1-1-0)과 동일한 요소 포함
- 추가 요소는 Creation Spread 2 참고

### 3. Creation Spread 2 (1-1-0): 상세 정보 추가 (스크롤 하단)

#### 추가 구성 요소
- **AI 자동생성 버튼**: 
  - 현재 미구현 (NotImplemented 표시)
- **텍스트 입력창**: 
  - 국가/지역, 품종
  - 기본 입력창 스타일 동일
- **드롭다운 + 텍스트 입력창**: 
  - 드롭다운 선택 시 텍스트 입력창에 자동 추가
  - "직접 입력" 선택 시: 내부적으로 ETC 처리, 사용자 입력 가능
  - 필드: 가공방식, 로스팅포인트, 추출방식
- **테이스팅 노트**: 
  - 쉼표 또는 띄어쓰기 단위로 해시태그 생성
  - 최대 5개
  - 해시태그 스타일:
    - 배경: `#2B1E1A`
    - 텍스트: `#FFFFFF`
    - Border Radius: 20px
    - 폰트: 30px, Weight: 400
- **기록하기 버튼**: 
  - createNote + createDetail 실행

### 4. Library Folded (1-0-0): 라이브러리 화면 (상세 필터 꺼짐)

#### 구성 요소
- **검색창**: 
  - 텍스트 입력
  - search 서비스 활용
  - 입력창 스타일 동일
- **상세필터 폴드**: 
  - 클릭 시 펼쳐짐 (library_spread로 전환)
  - 접힘 상태 표시
- **정렬 버튼**: 
  - 최신순, 별점순 선택 가능
  - 정렬 상태에 맞게 리스트 재구성
- **노트 컴포넌트**:
  - **basic 컴포넌트**:
    - 왼쪽: 카페명, 마신 날짜, 메뉴명, 한줄평
    - 오른쪽: 산미, 바디, 쓴맛 수치
    - 배경: `#FFFFFF`
    - 테두리: `rgba(90, 58, 46, 0.3)`
    - Border Radius: 10px
  - **detail_included 컴포넌트**:
    - basic 컴포넌트와 동일
    - 메뉴명과 한줄평 사이에 상세 정보 표시:
      - 국가/지역, 품종, 가공방식, 로스팅포인트, 추출방식 중 NULL이 아닌 항목만 표시
    - 해시태그: 오른쪽 하단에 나열
      - 해시태그 스타일: 배경 `#2B1E1A`, 텍스트 `#FFFFFF`, Border Radius 20px
- **기록하기 버튼** (FloatingActionButton): 
  - Creation (1-1-0) 팝업 호출
- **하단 내비게이션 바**: 
  - 라이브러리, 갤러리, AI 가이드 간 탭 전환
  - 현재 탭 강조 표시

### 5. Library Spread (1-0-0): 라이브러리 화면 (상세 필터 켜짐)

#### 추가 구성 요소 (library_folded와 공통 항목 제외)
- **상세 필터 영역**: 
  - 산미, 바디, 쓴맛 슬라이더 (1~10)
  - 각 슬라이더에 대한 라벨 표시
  - 배경: `#FFFFFF`
  - 테두리: `rgba(90, 58, 46, 0.3)`, 2px solid
  - Border Radius: 10px
  - similarity 서비스를 활용한 필터링 적용
  - 필터 값에 따라 리스트 자동 업데이트

## 상호작용 패턴

### 네비게이션
- **하단 내비게이션 바**: 항상 표시, 3개 탭 (LIST, GALLERY, AI 가이드)
- **FloatingActionButton**: 공통, 모든 화면에서 표시 (Creation 팝업 호출)
- **팝업**: Creation, Detail 화면은 팝업으로 구현 (depth 증가 없음)

### 스크롤
- 상세 정보가 많은 화면은 스크롤 가능
- 스크롤 영역 명확히 구분
- 스크롤 인디케이터 제공

### 피드백
- 버튼 클릭 시 시각적 피드백
- 입력 필드 포커스 표시
- 로딩 상태 표시 (필요 시)
- 에러 메시지 명확하게 표시

### 필터 및 검색
- 검색어 입력 시 실시간 검색 결과 업데이트
- 필터 적용 시 리스트 자동 갱신
- 필터 상태 시각적 표시 (활성화/비활성화)

## 구현 참고사항

### 데이터 구조
- **Note**: 기본 커피 노트 정보 (카페명, 메뉴명, 날짜, 한줄평, 산미/바디/쓴맛, 점수, 이미지)
- **Detail**: 상세 정보 (국가/지역, 품종, 가공방식, 로스팅포인트, 추출방식, 테이스팅 노트)

### 서비스 연동
- **NoteService**: 노트 CRUD, 검색, 정렬
- **DetailService**: 상세 정보 CRUD
- **Similarity Service**: 유사도 기반 필터링

### 상태 관리
- 화면별 상태 관리 필요
- 필터 상태, 정렬 옵션, 검색어 등 관리
- 데이터 새로고침 메커니즘
