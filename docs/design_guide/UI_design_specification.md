# creation (1-1-0): 생성 팝업 기본 화면
## 구성요소
- 뒤로가기 버튼 -> 이전 화면
- 이미지 추가하기 -> 이미지 업로드
- 텍스트 입력창 (메뉴명, 카페명, 날짜명, 한줄평)
- 슬라이더 (산미, 바디, 쓴맛): 1~10 단위
- 별점 입력: 이미지, CSS에는 없으나 기존 구현안 참고하여 사용할 것. (별 5개 클릭하여 입력하는 형태)
- 상세정보 추가하기 체크 박스: 체크할 경우, 아래가 스크롤할 수 있게 확장됨. 자세한 내용은 creation_spread, creation_spread_2 참고
- 기록하기: 입력한 정보를 바탕으로 createNote

## 구성요소

# creation_spread_2 (1-1-0): 생성 팝업에서 상세 정보 추가하기 버튼 눌렀을 때 (스크롤 하단)
## 구성요소 (상기 언급한 요소 제외하고)
- AI 자동생성 버튼: 현재는 구현하지 않음. 클릭하면 NotImplemented 뜨게 할 것
- 텍스트 입력창: 국가/지역, 품종
- 텍스트 입력창 + 드롭다운: 드롭다운에서 선택하면 해당 내용이 텍스트 입력창에 자동 추가됨. 직접 입력을 선택할 경우, 내부적으로는 ETC로 처리하며, 사용자가 텍스트 입력창에 내용 입력할 수 있게 함. (텍스트 입력창 -> *_text, 드롭다운 -> *)
- 테이스팅 노트: 쉼표 또는 띄어쓰기 단위로 해시태그 생성 (최대 5개)
- 기록하기 버튼: 입력한 정보를 바탕으로 createNote + createDetail

# library_folded (1-0-0): 라이브러리 화면 (상세 필터 꺼짐)
## 구성요소
- 검색창: 텍스트 입력 (search 서비스 활용)
- 상세필터 폴드: 클릭하면 펼쳐져서 library_spread로 전환
- 정렬 버튼: 최신순, 별점순에 맞게 정렬되도록 함.
- 컴포넌트
  - basic 컴포넌트: 카페명, 마신 날짜, 메뉴명, 한줄평이 왼쪽에 위치하고, 오른쪽에 산미, 바디 쓴맛 존재
  - detail_included 컴포넌트: 기존 basic 컴포넌트와 동일하되, 메뉴명과 한줄평 사이에 국가/지역, 품종, 가공방식, 로스팅포인트, 추출방식 중 NULL이 아닌 것을 모아서 디스플레이. 해시태그는 오른쪽 하단에 나열
- 기록하기 버튼: creation (1-1-0) 팝업 호출
- 하단 내비게이션 바: 라이브러리, 갤러리, AI 가이드 간 탭 전환
